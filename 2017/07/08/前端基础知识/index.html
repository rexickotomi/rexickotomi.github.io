渲染引擎(浏览器内核):
  决定了浏览器如何显示网页的内容以及页面的格式信息,是兼容性问题出现的根本原因。

我们在浏览器里看到的网页其实是由以下三层信息构成的一个共同体:
  结构层
    网页的结构层是由HTML或XHTML之类的标记语言负责创建
  表示层
    表示层由CSS负责完成，css描述页面内容应该如何呈现
  行为层
    行为层负责内容应该如何响应事件这一问题，这是JavaScript语言
    和DOM主宰的领域。

    虽然在这三种技术之间存在着一些潜在的重叠区域，但是，我们应该尽量让这三种技术各司
  其职，实现分离，远离如用dom脚本编程控制css样式这种”不务正业”的做法。
    此外，HTML关注的是结构（语义），css关注的是样式，结构和样式应该分离。应该把所有
  的格式化和设计相关的实现放在CSS文件中，让HTML只用来表示文档结构，不用于页面样式的
  设计中。

HTML、XHTML和HTML5
  HTML（超文本标记语言）指的是HTML 4.01，XHTML（扩展的超文本标记语言）是HTML的过渡版——XML风格的HTML 4.01，HTML 5是下一代的HTML——HTML 4.01的升级版。
  HTML5不再是单纯意义上的标签了，它已经远远超越了标签的范畴。HTML5除了新增部分标签之外，还增加了一组技术，包括canvas、SVG、WebSocket、本地存储等。这些新增的技术都是使用JavaScript来操作。也就是说，HTML5使得HTML从一门“标记语言”转变为一门“编程语言”。
 
  XHTML相对于HTML来说，在语法上更加严格。XHTML和HTML主要区别如下：
  1）XHTML标签必须闭合。
  2）XHTML标签以及属性必须小写。
  3）XHTML标签属性必须用引号。
  4）XHTML标签用id属性代替name属性。
 
  单纯从新增的标签上来看，HTML 5有以下几个特点：
  1）文档类型说明——<!DOCTYPE html>，被极大地简化了。
  2）标签不再区分大小写。
  3）允许属性值不加引号。
  4）允许部分属性的属性值省略，如<input type="checkbox" checked="checked" />可省略为
   <input type="checkbox" checked />

浏览器标题栏小图标
  想要在浏览器标题栏添加小图标，只需在head标签添加一个如下的link标签即可：
  <link rel="shortcut icon" type="image/x-icon" href="*.ico" />
  注：rel和type这两个属性的取值是固定形式，而href属性取值为小图标的地址，这个地址是根据小图标在站点文件夹路径而定的，跟图片引用路径类似。
